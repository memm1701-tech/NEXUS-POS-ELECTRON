/ Especificaciones Técnicas

Diseñado para la robustez absoluta

Cada línea de código de Nexus está optimizada para prevenir cuellos de botella en operaciones de alto volumen.

offline_bolt

Tolerancia a Fallos Offline

Si la red principal cae, el sistema de caché local permite continuar operaciones de consulta sin interrupción. La cola de sincronización se retoma automáticamente al restablecer conexión.

account_balance

Cumplimiento Fiscal

Arquitectura de base de datos preparada para las normativas del SENIAT. Registros inmutables, correlativos blindados y reportes Z generados con precisión criptográfica.

currency_exchange

Motor Multimoneda Dinámico

Actualización automática de Tasa BCV (o manual centralizada) que recálcula instantáneamente el inventario en toda la red de cajeras sin recargar las vistas.

inventory_2

Locking de Inventario Atómico

El servidor principal aplica bloqueos transaccionales a nivel de fila durante la facturación para evitar race conditions y sobreventas de productos en múltiples cajas.

/ Developer Experience

Diseñado con mentalidad API-First

Aunque funciona perfecto como una aplicación de escritorio, el backend de Nexus POS expone endpoints RESTful locales. Esto permite que el nodo Administrador procese peticiones de las cajeras de manera asíncrona y ultra-rápida.

  • check_circle Consultas no bloqueantes
  • check_circle WebSockets para notificaciones en tiempo real
  • check_circle Seguridad IPC (Inter-Process Communication)
cajera_request.js 
// Ejecutado en inicio_cajera.html antes de imprimir
async function procesarVenta(carrito) {
  try {
    const response = await fetch('http://nexus-admin:3000/api/facturar', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        items: carrito,
        caja_id: window.config.stationID
      })
    });

    const data = await response.json();
    if (data.exito) {
      // Imprime con el correlativo devuelto por el Admin
      imprimirTicket(data.correlativo_fiscal); 
    }
  } catch (error) {
    UI.showError("Error de comunicación con el nodo principal");
  }
}
